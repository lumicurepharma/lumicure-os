## Purpose

This guide keeps LOS readable, consistent, and useful for both humans and AI agents.

## Rules

1. Write in clear, direct language.
2. Prefer short sections over long essays.
3. Put decisions in Decision Logs, RFCs, or ADRs.
4. Keep strategy separate from implementation tasks.
5. Update metadata when a page changes.
6. Avoid hype language unless it is part of brand copy.
7. Define terms before using them as operating concepts.

## Page Metadata

Every LOS page must include:

- Title
- Version
- Status
- Owner
- Last Updated
- Dependencies
- Related Chapters
- Review Date
- References
- Decision History

## Naming

- Use numbered modules at the top level.
- Use descriptive page names.
- Use `DL-0000`, `RFC-0000`, and `ADR-000` numbering for records.
